How To Fix Violation
If rule score is N/A, then fulfill preconditions - current release is planned and unresolved user stories exist. To ensure this check that rules RP 1.1 - Define release and RP 1.2 - Create user stories are passed.
In order to fix violation project manager should assign at least one unresolved user story to current release or iterations which belong to current release.
Follow step-by-step instructions how to fix violation:
    1. Switch to the Projects perspective
    2. Select Work Items in the Topics tree
In order to assign user story to release or iterations which belong to current release make following:
    1. Select user story in unresolved state in the list of work items
    2. Click on the Edit button
    3. Select current release or iterations which belong to current release in the Time Point combo-box
    4. Click on the Save button
XP Theory
Before starting implementation of the new program or new version, team should define the list of new features and the time, which is required to implement them. This produces a release plan, which should be agreed by customers and development team on the release planning meeting. A set of created user stories is an input for release planning meeting. During the meeting customers select most important user stories from their point of view and assign them to release, taking into account estimation, risks and other factors.

Instructions
Release plan is considered as existed in Polarion, if at least one user story in unresolved state is assigned to the current release or iterations which belong to current release. Assignment user story to release is performed by defining release as a timepoint for user story. Preconditions for this rule are existence of unresolved user stories and current release.

How it works
This rule is automatically checked. During the check, unresolved user stories, assigned to current release or iterations which belong to current release are searched. If no one is found it means that release plan doesn't exist and a new violation is created and assigned to project manager.
