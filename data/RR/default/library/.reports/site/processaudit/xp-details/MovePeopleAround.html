How To Fix Violation
If rule score is N/A, then fulfill precondition - existence of source code. To ensure this check that rule PO 1.3 - Share project source code is passed. In order to fix violations engage more developers to work with violated package in order to become familiar with it and share this experience inside the team.
XP Theory
Cross training is often an important consideration in the way trying to avoid islands of knowledge, which are so susceptible to loss. Moving people around the code base in combination with pair programming does cross training. Instead of one person who knows everything about a given section of code, everyone on the team knows much of the code in each section.

Instructions
Encourage developers to work on a new section of the system at least part of each iteration. As the result every developer will work with different parts of the system and will understand it better.

How it works
This rule is automatically checked. During the check, maven project source code is analyzed to detect changes, which were performed for each class. Authors of the last five changes are detected for each class. This information used to define a list of active authors for each source code package. Active authors are those ones, who recently changed a least 20 percents of package content. If number of active authors is less then 2, it means that package has a lack of developers, who a familiar enough with it, so new violation is created and assigned to the project manager.
