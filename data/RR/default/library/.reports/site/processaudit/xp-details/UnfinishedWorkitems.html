How To Fix Violation
If rule score is N/A, then fulfill precondition - existence of passed iterations. If project doesn't reach the first iteration, it's a normal situation that the score is N/A.
In order to fix violations project manager should unassign unresolved work items from passed iterations. Follow step-by-step instructions how to fix violation:
    1. In the violation list click on reference to the unresolved work item, assigned to passed iteration
    2. Click on the Edit button
    3. Select empty value in the Time Point combo-box
    4. Click on the Save button
XP Theory
An iteration it isn't only a defined date, but also iteration plan, which consists of work items, assigned to the iteration. When iteration date is passed, an iteration plan becomes not relevant. Ideally, till the iteration date all planned work items should be completed, but it's not always so in reality. In this case unfinished work items should be included to the next iteration planning, but they are still assigned to passed iteration. In this case as soon as iteration date is passed, all unfinished work items should be unassigned from passed iteration.

Instructions
Project manager, who is responsible for iteration planning, should unassign unresolved work items from the iteration except work item type of Requirement, as soon as iteration date is passed. After that work items can be included to the next iteration during iteration planning meeting. Assigning/unassigning work items to iterations performed by changing Timepoint field.

How it works
This rule is automatically checked. During the check, passed iterations are verified to find unresolved work items except work type of Requirement , assigned to them. If such work items are found, then new violations are created and assigned to project manager, which is responsible for iteration planning.
