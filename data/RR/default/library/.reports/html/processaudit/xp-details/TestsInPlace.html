How To Fix Violation
If rule score is N/A, then fulfill precondition - existence of the source code for maven project. To ensure this check that rule PO 1.3 - Share project source code is passed.
In order to fix violation JUnit tests source code should be committed to the repository.
Also configure Maven descriptor to point path to source directory with tests, if needed. On order to do it in Maven descriptor in build section configure/add testSourceDirectory element which points to test directory.
Run build process and be sure that tests are passed during it. Follow step-by-step instructions in order to run build process for project in Polarion:
    1. Switch to the Projects perspective
    2. Select needed project from project list
    3. Select Builds in the Topics tree
    4. Click on Create Build button, where you see Create New Build window
    5. Fill correct build attributes and run build by clicking on Finish button
XP Theory
Automatic unit testing is the one of the most effective ways to guarantee code quality even from the early project stages. It is based on the two principles: source code is covered by unit tests, which verifies that it works correctly; and tests are executed automatically each day. As the result source code is covered by tests and team knows at any moment how quality it is.

Instructions
In the Polarion unit tests are based on JUnit framework. Team should write JUnit tests and commit them to the repository. These tests are automatically executed in scope of automatic build process. Polarion uses Maven 2 infrastructure for build process organization, so for automatic tests execution path to tests should be specified in the Maven descriptor. If project consist of sub-projects, then unit tests should be executed for each subproject.

How it works
This rule is automatically checked. During the check, unit tests results are searched. If they can't be found, it means that automatic tests don't exist, so new violation is created and assigned to the project manager, who should organize automatic unit testing in the project.
