How To Fix Violation
If rule score is N/A, then fulfill precondition - current release is planned. To ensure this check that rule RP 1.1 - Define release is passed.
In order to fix violation project manager should schedule one or many iterations between previous and current release.
Follow step-by-step instructions how to fix violation:
    1. Switch to the Administration perspective
    2. Select Work Items > Time Points in the Topics tree
In order to create a new iteration timepoint make following:
    1. Click on the Create New Timepoint button
    2. Enter a special identifier, which starts from Release, select the date later then current and fill other fields
    3. Click on the Create button
XP Theory
Iterative development adds agility to the development process. A release schedule is divided to a set of shorter periods, which are called iterations. At the end of each iteration development team delivers running useful program with a set of new features. Short cycles between deliveries allow keeping program in running state and simplify feedback collecting from customers and target users.

Instructions
Upcoming release should be splitted by project manager to iterations. In the Polarion iterations are represented by timepoints, which should have special identifier, which starts from Release. At least one such timepoint should be created between previous and current releases. Precondition for this rule is existence of current release.

How it works
This rule is automatically checked. During the check, timepoints with dates between previous release (or project start) and current release, are searched. If no one is found it means that release doesn't splitted to iterations and a new violation is created and assigned to project manager.
