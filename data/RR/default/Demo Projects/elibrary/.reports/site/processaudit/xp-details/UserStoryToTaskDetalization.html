How To Fix Violation
If rule score is N/A, then fulfill precondition - iteration plan exist. To ensure this check that rule IP 2.1 - Create iteration plan is passed.
In order to fix violation user story should be linked with one or few correspondent tasks. Links should be defined for tasks, should have implements role and refer parent user story.
Follow step-by-step instructions how to fix violation:
    1. Switch to the Projects perspective
    2. Select Work Items in the Topics tree
If there are no derived tasks for user story, they should be created:
    1. Click on the Create button and select Task from the popup menu
    2. Fill required fields
    3. Click on the Create button
In order to link user story and task make following:
    1. Select task in list of work items
    2. Click on the Edit button
    3. Select Requirement in the Linked Work Items > Title area and select implements role
    4. Click on the Add button to add a link
    5. Click on the Save button
XP Theory
On the iteration planning meeting customers select user stories, which should be completed during the iteration. Requirements and additional useful information about each selected user story is provided by customers to developers face-to-face. As the result user stories are clarified enough to formulate a set of development tasks, which should be performed. Tasks are atomic amount of work, which can be precisely estimated and used to create iteration plan.

Instructions
User stories, which were selected to iteration, should be unassigned from release and assigned to iteration by changing Timepoint field for correspondent work item. In the iteration plan they are represented by tasks, which derived from user stories. Relation between user stories and tasks can be exposed in Polarion as work items linking. Thus, tasks should have implements link role to user story work item. Preconditions for this rule are existence of iteration plan.

How it works
This rule is automatically checked. During the check, unresolved user stories, assigned to current iteration, are checked. If there are user stories without links to correspondent tasks, then new violations are created and assigned to authors of user stories. Link between user story and task considered as valid only if it is defined for task and has implements role.
