How To Fix Violation
If rule score is N/A, then fulfill precondition - existence of automatic tests. To ensure this check that rule QC 1.1 - Create automatic tests is passed.
In order to fix violation check what source code packages have not enough direct tests coverage and create dedicated tests for methods, which are not tested directly.
When it will be finished for all violations, then you can run build again to ensure that direct tests coverage was increased. Follow step-by-step instructions how to run build:
    1. Switch to the Projects perspective
    2. Select needed project from project list
    3. Select Builds in the Topics tree
    4. Click on Create Build button, where you see Create New Build window
    5. Fill correct build attributes and run build by clicking on Finish button
XP Theory
Percent of the source code coverage by automatic tests is not the main measure of the automatic tests quality. All source code methods, which are executed in scope of tests, are treated as covered by tests regardless if methods are called directly by tests or by other methods. If methods are called directly by tests it means that tests are dedicated to evaluate these methods, which suppose more relevant testing result. Ideally all methods should have dedicated tests.

Instructions
When developers create automatic tests, they should create dedicated tests for each source code method. It allows controlling how method works and catches more problems on automatic testing.

How it works
This rule is automatically checked. During the check, automatic tests execution results are evaluated. For each package, which has direct classes, percent of public methods covered by tests, i.e. executed in scope of testing, is calculated. If coverage percent is less then 50, then violation is created and assigned to project manager, who is responsible for the quality of automatic testing.
