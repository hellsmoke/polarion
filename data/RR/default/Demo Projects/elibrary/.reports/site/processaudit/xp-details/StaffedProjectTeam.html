How To Fix Violation
In order to fix violation, assign persons to the unassigned role. Follow step-by-step instructions how to fix violation:
    1. Switch to the Administration perspective
    2. Select User Management > Roles topic
If role is missed in the Polarion configuration, then create it:
    1. Click on the Create New Role button
    2. Enter the role name
    3. Click on the Create button
Assign users for role:
    1. Click on the Edit button for role
    2. Select user from Users > Name combo-box and click on the Add button
    3. Click on the Save button
XP Theory
In order to start new project you should have a team, which will work on it. This team should consists of developers, who will perform program implementation, project manager, who will control the project progress and manage the team and customers, assigned to the project, which define requirements and check program functionality.

Instructions
In the Polarion team is assigned to the project through the administrative interface, which allows to define particular role or a few roles for each person. Thus, project manager should have project_admin identifier role, developers should have project_developer identifier role and customers should have project_customer identifier role assigned.

How it works
This rule is automatically checked. During the check, roles defined for project are checked. If at least one of the roles project_admin identifier, project_developer identifier and project_customer identifier has no assigned persons, then new violation is created and assigned to project lead, which is responsible to staff management.
