How To Fix Violation
If rule score is N/A, then fulfill precondition - existence of source code. To ensure this check that rule PO 1.3 - Share project source code is passed. In order to fix violation developers should make often source code commits to the repository, not less then once per 1 days.
XP Theory
Developers should be integrating and releasing code into the code repository every few hours, when ever possible. In any case never hold onto changes for more than a day. Continuous integration often avoids diverging or fragmented development efforts, where developers are not communicating with each other about what can be re-used, or what could be shared.

Instructions
Developers should commit the source code into repository often, usually after each task completion, but not less then once per day. It guarantees that source code changes are propagated to all team members soon and integration problems can be reduced.

How it works
This rule is automatically checked. During the check, project source code is analyzed to detect dates of recent changes. If there are no changes were made in last 1 days, taking into account day offs, it means that project code isn't integrated often, so new violation is created and assigned to the project manager.
