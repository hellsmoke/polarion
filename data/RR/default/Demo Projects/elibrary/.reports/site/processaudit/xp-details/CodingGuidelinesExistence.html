How To Fix Violation

XP Theory
In order to make the source code standardized, development team should agree guidelines for code styling, formatting and used development approaches. This is formulated in coding guidelines document, which should be placed to accessible place. All team members should follow guidelines, described in this document.

Instructions
In the Polarion coding guidelines should be formalized as the document stored in the repository. Team members use this document as the reference for code styling, formatting and designing.

How it works
This rule is automatically checked. During the check, coding guidelines document: guidelines.doc is searched in the repository. If it is missed, the the new violation is created and assigned to the project manager, who is responsible to control this rule.
