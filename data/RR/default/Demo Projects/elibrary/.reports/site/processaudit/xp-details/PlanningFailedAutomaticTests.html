How To Fix Violation
If rule score is N/A, then fulfill precondition - existence of current iteration. To ensure this check that rule IP 1.1 - Define iteration is passed.
In order to fix violation project manager should schedule failed automatic tests to current or next iteration. Follow step-by-step instructions how to do that:
    1. Click reference to failed automatic test in the violation list
    2. Click on the Edit button
    3. Select current or next iteration in the Time Point combo-box
    4. Click on the Save button
XP Theory
Automatic tests are extremely helpful to control the quality of the program. Written once, they continuously executed to notify about any changes in program behavior. Problems, which are found using automatic tests, should be processed with priority. Even if problem isn't a bug, but expected change in behavior, automatic tests should be changed as soon as possible to reflect this. Project manager should care about proper planning of automatic tests failures.

Instructions
In the Polarion failed automatic tests are represented as work items with type Defect, which field unit test is filled to link with automatic test. If there are failed automatic tests in unresolved state, they should be scheduled by project manager to current or next iteration.

How it works
This rule is automatically checked. During the check all unresolved work items with type Defect are verified. If they are linked with automatic tests, then they should be scheduled to current or next iteration, otherwise new violations are created and assigned to project manager, who is responsible for iteration planning.
