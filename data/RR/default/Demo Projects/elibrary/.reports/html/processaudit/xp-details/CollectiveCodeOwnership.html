How To Fix Violation
In order to fix violation, the source code for projects should be committed to the repository. It doesn't allow committing empty source folder. After committing Maven 2 descriptor should be updated in order to correctly point to the source folder location.
XP Theory
The project source code doesn't own personally by authors or project manager or someone else. The source code is public and shared and any line of the code can be changed by any team member. This principle creates open style of development when technical decisions are made by entire team and entire team has a vision about program architecture.

Instructions
In the Polarion source code can be shared between all team members through repository, so each team member have overall source code and can change it if required in any moment.

How it works
This rule is automatically checked. During the check, the maven projects source code location is evaluated. In case of multi-project layout, maven subprojects source code locations are evaluated. If there isn't at least at one maven project shared source code, then new violation is created and assigned to the project manager.
