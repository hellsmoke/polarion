How To Fix Violation
In order to fix violation project manager should create a timepoint, which should have special identifier, which starts from Release and date later then current date.
Follow step-by-step instructions how to fix violation:
    1. Switch to the Administration perspective
    2. Select Work Items > Time Points in the Topics tree
In order to create a new release timepoint make following:
    1. Click on the Create New Timepoint button
    2. Enter special identifier, which starts from Release, select the date later then current and fill other fields
    3. Click on the Create button
XP Theory
The development team needs to release iterative versions of the program to the customers often. This is critical to getting valuable feedback in time to have an impact on the further development process. Before start of new program version development, customers together with project manager define next release date. Customers and project manager can select some particular date as a release date. For example, they can decide to release new version in two month. Or they can plan to include predefined list of new features to the new version and in this case a release date is defined by a time for features implementation. In any case a time between two releases shouldn't be too short or too long.

Instructions
Each project should have an upcoming release scheduled by the project manager. In the Polarion releases are represented by the timepoints which should have a special identifier, which starts from Release.

How it works
This rule is automatically checked. During the check, timepoints which should have special identifier, which starts from Release and dates greater then current, are searched. If no one is found it means that release doesn't planned and a new violation is created and assigned to project manager.
