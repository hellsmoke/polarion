How To Fix Violation
If rule score is N/A, then fulfill precondition - splitting current release to iterations. To ensure this check that rule RP 2.3 - Split release to iterations is passed.
In order to fix violations, project manager should unassign tasks from advance scheduled iterations. Follow step-by-step instructions how to do that:
    1. In the violation list click on reference to the work item scheduled to advance scheduled iteration
    2. Click on the Edit button
    3. Select empty value in the Time Point combo-box
    4. Click on the Save button
XP Theory
In order to be effective, detailed planning should be short-term. It doesn't make sense to schedule tasks on few iterations ahead. If something will be changed, you need to change plan for all upcoming iterations. Instead have an iteration planning meeting at the beginning of each iteration to plan out what will be done. Just-in-time planning is an easy way to stay on top of changing user requirements.

Instructions
Make planning, i.e. assign work items, only to the current or next iteration, but not to the following iterations. Assigning work items to iterations performed by selecting iteration in Timepoint field of work item.

How it works
This rule is automatically checked. During the check, current release iterations after the next iteration, are verified. If they have assigned unresolved work items, then new violations are created and assigned to project manager, who is responsible for iteration planning.
