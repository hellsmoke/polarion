MC 8 - Perform sequential integration
XP Theory
Only one development pair integrates, tests and releases changes to the source code repository at any given moment. Single threaded integration allows a latest version to be consistently identified. This is not to imply that you can not integrate your own changes with the latest version at your own workstation any time you want. You just can't release your changes to the team with out waiting for your turn.

How it works
This rule should be checked manually. Read instruction and follow it.
