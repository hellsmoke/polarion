How To Fix Violation
If rule score is N/A, then fulfill preconditions - current release plan exists. To ensure this check that rule RP 2.1 - Create release plan> is passed.
In order to fix violation project manager should assign more unresolved user stories to current release.
Follow step-by-step instructions how to fix violation:
    1. Switch to the Projects perspective
    2. Select Work Items in the Topics tree
In order to assign user story to release make following:
    1. Select user story in unresolved state in the list of work items
    2. Click on the Edit button
    3. Select current release in the Time Point combo-box
    4. Click on the Save button
XP Theory
During release planning meeting team should select right number of user stories to include into release. It doesn't makes sense to leave a lot of free space, instead define realistic plan and change it if something will go wrongly.

Instructions
Project manager should care about assigning right number of user stories for the release. Total estimates of user stories should not be less then release duration, otherwise release plan can't be defined as good.

How it works
This rule is automatically checked. During the check, total estimation of user stories, assigned to release, is calculated. This value is compared with duration of release, taking into account only working days. If total estimation is less then release duration on more then 20 percents, it means that release plan contains too less user stories, so new violation is created and assigned to the project manager.
