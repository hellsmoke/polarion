XP Theory
Automatic tests it's an effective way to control the quality of the program. Developed once, automatic tests don't require human efforts for execution, so they can be scheduled to work each day. As the result at any moment team have actual information about the functionality, which work and which fail. Automatic tests are targeted at verifying the source code, so should pass the evolution together with the source code. Team should keep an eye on failed tests and fix the code or change the test to achieve passing.

Instructions
In the Polarion unit tests are based on JUnit framework. Team should write JUnit tests and commit them to the repository. These tests are automatically executed in scope of automatic build process. Team should care about successful passing of all unit tests without errors and failures.

How it works
This rule is automatically checked. During the check, automatic tests execution results are evaluated. If test execution finished unsuccessfully with error or failure, then new violation is created and assigned to the automatic test author.
