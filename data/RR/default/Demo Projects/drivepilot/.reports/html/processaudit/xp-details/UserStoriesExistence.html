How To Fix Violation
In order to fix violation project manager should arrange creation of new work items with Requirement type by customers.
Follow step-by-step instructions how to fix violation:
    1. Switch to the Projects perspective
    2. Select Work Items in the Topics tree
In order to create a new user story work item make following:
    1. Click on the Create button and in popup menu select Requirement item
    2. Fill work item fields
    3. Click on the Save button
XP Theory
Instead creation of large requirements documents at start of the new version or new program creation customers write a set of user stories. User stories contain information about features, which program should do. User stories are similar to usage scenarios, except that they are not limited to describing a user interface. User stories should only provide enough detail to make a reasonably low risk estimate of how long the story will take to implement. When the time comes to implement the story developers will go to the customer and receive a detailed description of the requirements face to face.

Instructions
Each project should have a set of user stories in unresolved state. In the Polarion user stories are stored in the Tracker as work items with Requirement type. Project should have at least one unresolved work item with type Requirement.

How it works
This rule is automatically checked. During the check, work items with Requirement type and empty Resolution field are searched. If no one is found it means that project doesn't have unresolved user stories and a new violation is created and assigned to project manager.
