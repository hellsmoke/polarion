How To Fix Violation
If rule score is N/A, then fulfill precondition - existence of passed releases. If project doesn't reach the first release, it's a normal situation that the score is N/A.
In order to fix violations project manager should unassign unresolved user stories from passed releases. Follow step-by-step instructions how to fix violation:
    1. In the violation list click on reference to the unresolved user story, assigned to passed release
    2. Click on the Edit button
    3. Select empty value in the Time Point combo-box
    4. Click on the Save button
XP Theory
A release it isn't only a defined date, but also release plan, which consists of user stories, assigned to the release. When release date is passed, a release plan becomes not relevant. Ideally, till the release date all planned user stories should be completed, but it's not always so in reality. In this case unfinished user stories should be included to the next release planning, but they are still assigned to passed release. In this case as soon as release date is passed, all unfinished user stories should be unassigned from passed release.

Instructions
Project manager, who is responsible for release planning, should unassign unresolved user stories from the release, as soon as release date is passed. After that user stories can be included to the next release during release planning meeting. Assigning/unassigning user stories to releases performed by changing Timepoint field of correspondent work item to release or one of it iterations.

How it works
This rule is automatically checked. During the check, passed releases are verified to find unresolved user stories, assigned to them or one of their iterations. If such user stories are found, then new violations are created and assigned to project manager, which is responsible for release planning.
