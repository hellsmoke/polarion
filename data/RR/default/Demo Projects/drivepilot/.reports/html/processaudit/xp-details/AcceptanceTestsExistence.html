How To Fix Violation
In order to fix violation project manager should arrange creation of new work items with Test Case type.
Follow step-by-step instructions:
    1. Switch to the Projects perspective
    2. Select Work Items in the Topics tree
In order to create a new acceptance test work item make following:
    1. Click on the Create button and in popup menu select Test Case item
    2. Fill work item fields
    3. Click on the Save button
XP Theory
During iteration when developers work on the user stories implementation, customers create acceptance tests. The purpose of acceptance tests is to verify that user story's functionality works as expected. Acceptance tests are black box tests, which represent some expected results from the program.

Instructions
In Polarion acceptance tests are stored as work items with Test Case type. Project should have at least one unresolved work item with type Test Case.

How it works
This rule is automatically checked. During the check, work items with Test Case type and empty Resolution field are searched. If no one is found it means that project doesn't have unresolved acceptance tests and a new violation is created and assigned to project manager.
