How To Fix Violation
If rule score is N/A, then fulfill precondition - existence of current iteration and Acceptance Tests. To ensure this check that rules IP 1.1 - Define iteration and QC 1.2 - Create acceptance tests are passed.
In order to fix violation project manager should schedule failed acceptance test to current or next iteration. Follow step-by-step instructions how to do that:
    1. Click reference to failed acceptance test in the violation list
    2. Click on the Edit button
    3. Select current or next iteration in the Time Point combo-box
    4. Click on the Save button
XP Theory
Acceptance tests are created from user stories to check that user stories were implemented correctly. Passing acceptance tests is indispensable condition to consider user stories as completed. Usually acceptance tests are performed at the end of iteration to verify and accept new features. Acceptance tests are also used as regression tests prior to a production release. Acceptance tests failures point at important problems in functionality, so should be fixed as soon as possible. Project manager is responsible to schedule these failures to closest iteration.

Instructions
Acceptance tests are represented in the Polarion by work items with Test Case type. Acceptance tests failures are stored in the Polarion as work items with type Defect, which linked with correspondent acceptance test. Thus, acceptance tests failures should have relates to link role to acceptance test work item. If there are failed acceptance tests in unresolved state, they should be scheduled by project manager to current or next iteration.

How it works
This rule is automatically checked. During the check all unresolved work items with type Defect are verified. If they are linked with Test Case, then they should be scheduled to current or next iteration, otherwise new violations are created and assigned to project manager, who is responsible for iteration planning.
