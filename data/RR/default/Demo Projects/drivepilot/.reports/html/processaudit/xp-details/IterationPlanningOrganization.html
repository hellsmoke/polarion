How To Fix Violation
If rule score is N/A, then fulfill preconditions - current iteration is planned, unresolved user stories and tasks exist. To ensure this check that rules IP 1.1 - Define iteration, RP 1.2 - Create user stories, and IP 1.2 - Create tasks are passed.
In order to fix violation project manager should assign at least one unresolved user story and one unresolved task to current iteration.
Follow step-by-step instructions how to fix violation:
    1. Switch to the Projects perspective
    2. Select Work Items in the Topics tree
In order to assign user story or task to iteration make following:
    1. Select user story or task in unresolved state in the list of work items
    2. Click on the Edit button
    3. Select current iteration in the Time Point combo-box
    4. Click on the Save button
XP Theory
On the iteration meeting customers select user stories from the release plan, which should be completed during iteration. Customers should select first of all those user stories, which are most valuable from their point of view. Selected user stories are assigned to iteration and broken down to development tasks, which should be completed during iteration. As the result, iteration plan consist of user stories and derived tasks.

Instructions
Iteration plan is considered as existed in Polarion, if at least one user story and one task in unresolved state are assigned to the current iteration. Assignment user story and tasks to iteration is performed by defining iteration as a timepoint for work items. Preconditions for this rule are existence of current iteration, unresolved user stories and unresolved tasks.

How it works
This rule is automatically checked. During the check, unresolved user stories and tasks, assigned to current iteration are searched. If there are no such user stories or tasks it means that iteration plan doesn't exist and a new violation is created and assigned to project manager.
