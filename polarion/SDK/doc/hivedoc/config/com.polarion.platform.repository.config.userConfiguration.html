Schema
Element <config>
Id of configuration (used for retriving the configuration information) The configuration will be merge default and user specific configurations. Path in default repository to retrieve or store configuration information for a particular user. The path must contain substring [USER_ID] that will be replaced be id of the user for which the configuration is read or stored. Path in default repository to retrieve or store default configuration information. If omitted, it is assumed that this user configuration has no global default; attempts to read or write such configuration with no specified user will fail. The file processor generate suitable Object for configuration consumer, if omited the service will use the default. Default file processor returns the org.w3c.dom.Document as result of configuration processing. The service must implement the com.polarion.platform.repository.config.IDataHandler interface
