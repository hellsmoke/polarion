Schema
Element <config>
Id of configuration (used for retriving the configuration information) Hierarchic config will be looked in parent contexts if will be absent in the specific Assume hierarchic == true. The Configuration service will be marge all found configs from specific context up to the root. The relative path (according context location) to retriev or store configuration information. If omited the service will assume that configuration is located in the context root in the ${id}.config file. The file processor generate suitable Object for configuration consumer, if omited the service will use the default. Default file processor returns the org.w3c.dom.Document as result of configuration processing. The service must implement the com.polarion.platform.repository.config.IDataHandler interface
