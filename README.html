Polarion ALM 17 README File

Thanks for using or evaluating this Polarion product. This is the README file for version 17 of the Polarion ALM product family. Unless otherwise noted, the information is applicable to:

  • Polarion ALM
  • Polarion REQUIREMENTS
  • Polarion QA

The navigation menu above leads to information in this file.

About This Release

This is a major version release that rolls up features and enhancements released to customers during the last several Service Releases, and adds some new features and enhancements, including a first iteration of a new Resource Traceability feature, which enables linking Work Items to specific blocks in source code that implement them. It also adopts different version numbering for consistency with other products in the Siemens PLM Software product line. To learn more about what's in the release, please read the blog article: Polarion ALM 17 – What's New and Noteworthy.

  • If you are interested in knowing what specific tracker issues have been addressed since the last release, a report of Resolved Issues is published online. You can find the link on the product download pages.
  • If you are interested in the development history of the product(s) you are using, check the Version History file bundled with each product distribution.

License Compatibility

If your maintenance subscription is current, you can update from version 2015 to version 2016 and Service Releases using the License Key Code you already have. You only need to activate your updated installation, which you can do online or offline. For details, see the bundled HOW_TO_INSTALL_THIS_UPDATE.txt in the update distribution.

If you want to update from version 2014 (including Service Releases) or earlier, you will need to obtain a new license key. You can request it online at https://polarion.plm.automation.siemens.com/getlicense.

If you have questions about, or need help with licensing issues please contact the Technical Support team via one of the options listed HERE.

Additional Notes for Microsoft Internet Explorer

  • When using Internet Explorer on Windows 8.1, running Windows 8.1 in the desktop mode is recommended. Polarion's Flash-based features (LivePlan, etc.) may not work, and various dialogs as well as Clipboard access may be blocked if the Windows 8 style user interface (sometimes called "Metro") is used.
  • Polarion can be installed on Windows 8.1 for evaluation purposes. In order to use Internet Explorer, it may be necessary to define http://localhost (and http://127.0.0.1 for Metro mode) as an intranet site in: Tools > Internet Options > Security tab > Local Intranet > Sites > Advanced.

  • If running Internet Explorer on Windows Server 2012 (e.g. for testing):

    • If prompted by the browser, you will need to add "about:blank" as a trusted server.
    • You may need to add the local Polarion server and any remote Polarion server as trusted servers.
    • You need to enable javascript for the Polarion server

    (The above points do not apply when accessing Polarion running on Windows Server 2012 from other clients)

  • Users of Internet Explorer are recommended to specify their Polarion server in the Local Intranet security zone while ensuring that Compatibility View is not used.
  • There can be performance issues on client computers running Internet Explorer due to browser memory requirements. Occasional restarts of the browser are recommended.

Integrated Database Replaced for All Installations

In version 2015 SR2, Polarion replaced the integrated H2 database with PostgreSQL in new installations, and continues this with the current release.

In version 2016 and Service Releases, the update package for contains installation scripts that automatically install PostgreSQL to existing installations and update the system configuration to use PostgreSQL. Before updating a pre-2016 system to this release, please be sure to read the information in the Configuration.txt file, bundled in the update distribution.

Pre-2014 "Multi-Instance" Topography No Longer Supported

Several versions prior to version 2014 supported a topography of multiple Polarion instances which was called "Multi-Instance" setup, and it continued to work in Version 2014 and subsequent versions. It is no longer supported in Version 17. If you still use this setup, your administrator should review the Configuration.txt file bundled in the update distribution for Version 17 before updating your system.

32-bit Windows, Windows 7 No Longer Supported

Polarion no longer supports installation on 32-bit Windows systems. Bundled Apache, Java, Subversion, and PostgreSQL are all 64-bit versions in the installation archives for Windows. A 64-bit OS is now required, and the installer will fail with a message if run on 32-bit Windows. The main potential for impact is on evaluations, for which some people would use 32-bit Windows.

Polarion no longer supports Windows 7 for evaluation installations. Versions 8.1 and 10 are supported.

Java 8 Required

Polarion no longer supports Java versions prior to version 8. For more information, see the installation guide document for your OS, and the bundled Configuration.txt file.

Impact of Java 8 on JavaScript Scripts

Some JavaScripts may not work after updating to Java 8, because the JavaScript engine in Java 8 is not fully backwards compatible. It is recommended to migrate your scripts according to the migration manual: https://wiki.openjdk.java.net/display/Nashorn/Rhino+Migration+Guide. However, we provide you the possibility to temporarily run scripts in "Java 7 compatible" mode by setting the property com.polarion.scripting.useRhinoJsEngine=true in the polarion.properties system configuration file.

SLES 11 Service Pack 3 Required

New Polarion installations on SUSE Linux Enterprise Server (SLES) 11 require that Service Pack 3 be installed, or the installation will fail. Existing installations running on SLES 11 with Service Pack 2 can be updated. However, updating SLES 11 to Service Pack 3 is recommended.

Installation Documentation

New Installations

PDF documentation outlining relevant system requirements and installation procedures is bundled in the root folder of all installation archives. The Installation Guides for Windows and Linux are also provided as separate downloads on the Guides and Manuals page on the Polarion Software web site.

Customers who want to use Polarion in a clustered, multi-server environment should download the Polarion Enterprise Setup Guide (PDF), available from the Guides and Manuals page of the Polarion Software web site, in addition to the Installation Guide for the relevant operating system(s).
